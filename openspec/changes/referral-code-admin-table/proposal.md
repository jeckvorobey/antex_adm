## Why

В админке AntEx нужно явно управлять пользователями без реферального кода: показывать кнопку массовой генерации только когда это действительно нужно, а также отображать в таблице пользователей ключевые реферальные поля без лишней ручной работы.

## What Changes

- В таблице пользователей админки появляются колонки `referral_code`, `% начислений` и `balance`.
- Кнопка «Создать реферальный код» показывается только если есть хотя бы один пользователь с `referral_code = null`.
- Админка получает backend-контракт для запуска массовой генерации кодов и отображения состояния.
- UI таблицы приводится к аккуратному, читаемому виду без поломки текущих сценариев.

## Capabilities

### New Capabilities
- `admin-referral-users-table`: отображение referral code, referral percent and balance in user table.
- `admin-referral-code-generation-action`: conditional bulk generation action for null referral codes.

### Modified Capabilities
- `admin-user-management-ui`: уточняется состав колонок и условия показа action-кнопки.
- `admin-referral-data`: уточняется контракт данных для таблицы пользователей.

## Impact

- Страницы/таблицы пользователей в админке.
- Контракт с backend для данных таблицы и состояния кнопки.
- Тесты на видимость кнопки и наличие новых колонок.
